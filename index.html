🧠 Problem Statement: Local Food Finder UI

Design a homepage UI for a fictional food delivery service called LocalFoodie that helps users find local cuisine.

The design should be clean, intuitive, and user-friendly. Static data can be used; no API required.

✅ Checklist / Features to Implement (10 Checkpoints)

  1. Header Section with Branding: Logo or app name, and nav links (Home, Explore, Contact)
  2. Hero Section: Tagline (e.g., "Discover the taste of your neighborhood!") with a CTA like "Explore Now."
  3. Search Bar: Search for food or restaurants (bonus: city/area filtering)
  4. Cuisine Categories: Popular cuisines shown as clickable cards
  5. Sorting Feature: Sort by rating, price, delivery time
  6. Responsive Design: Should work on mobile, tablet, and desktop
  7. Dark/Light Mode Toggle (Bonus): Toggle for switching themes
  8. Footer Section: Social links, contact info, small About section
  9. Search Filter Results (Optional Bonus): Filter cards by category or search input
  10. Favorites Feature (Bonus): Users can mark restaurants as favorites using a heart icon or button

Tech Stack Allowed: HTML, CSS, JavaScript (Vanilla or frameworks like React, Vue.js, Svelte, or Astro). Tailwind CSS / Bootstrap are optional. No backend or API needed.

📜 Rules & Regulations

  1. Individual or team participation (max 3 members)
  2. Only frontend tools allowed: HTML, CSS, JavaScript (React, Tailwind allowed)
  3. Time limit: 120 minutes
  4. Use of AI (e.g. ChatGPT, Copilot) is allowed
  5. Use of GitHub for copying code or cloning repositories is strictly prohibited. Any form of plagiarism, including copying existing projects or code from GitHub, will result in disqualification.
  6. No use of pre-made templates
  7. Documentation must be submitted along with the project explaining the components and approach
  8. You can bring your own laptops; if unavailable, you may use the computer lab systems provided
  9. After the event, the submitted GitHub repository must be made public
